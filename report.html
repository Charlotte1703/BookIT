Report

Features of all pages:

Bootstraps
- Used to create the page layouts (from W3Schools)
Logo
- Displayed at top-left of each page, with the site name added as a separate header so it can be seen on smaller devices (this did not work with a banner logo)
- Links back to the homepage from all pages (there is still also a separate button for this)
Menu Buttons
- Consistently placed across all pages
- Colour changes to show which page is “active” (which page is currently being viewed)
- On smaller devices the buttons are hidden behind a menu button which will provide the user with a drop-down menu from which they will then be able to select the page that they would like to view next (the “active” feature still works here)
Ad Column
- Located on the left-hand side of each page, but at the bottom of each page on smaller devices
- Two books are “featured” on each page, each linking to the catalog page where the user is able to see the entire selection available
Footer
- Contains my name, N number, and email address
CSS
- The CSS for each page is located within the HTML code because of how the bootstraps are laid out – I did attempt to add them in separately and would have found this easier, but the bootstraps already contained a link for a stylesheet and it was not clear as to how a second style sheet could be added at this point.

Homepage

- Contains a brief description of what the website is and does and also includes two hyperlinks (one to the catalog page, and one to the order form page) so that the user can take a look at them.

Catalog Page (XML)

- This page contains a table which contains book data obtained from a separate XML page.
- The information displayed in the table for 10 IT-related books includes; title, image, author, publisher, ISBN, price, and also order code. All information except from the order codes was found on the Waterstones website.
- Images are stored in an “images” folder where they can easily be located and identified
- Table is dynamic and resizes with the browser window
- All of the book data has been used from the Waterstones website

Order Form

- Small description of how to use the page with headers to split the form up into the 3 parts.
- The form contains: drop-down menus containing the order codes, number fields with limited input (1-10), text fields, text fields with patterns to restrict input and keep it sensible, and date field limiting the minimum date to 1st January 2017
- “Place order” button – when the form is properly filled in, this button will allow the user to access the “submit” page

Submit

- No menu button for this page because it should not be accessible without filling in the order form correctly.
- Provides message for the user to thank them for their order and tell them that they will receive further information regarding estimated delivery dates.
- User also given a “unique order number” which is just a randomly-generated 10-digit number.

Slideshow

- Used HTML, CSS, JavaScript
- For this I created 4 banners to feature some of the books for sale on the site with some words of encouragement such as “learn a new skill…”
- Slideshow is dynamic to fit all devices and browser sizes
- Slides are on a timer and will keep on looping. Current slide number can be seen in top left corner, and also in the form of dots just below the slidesh
